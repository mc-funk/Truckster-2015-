Getting Started

This example web app was generated for you when you created your ExoHack app. You are not required to use it of course, but it provides a good base to build on, as well as some built-in Angular.js directives to handle common tasks, such as displaying realtime data from a device, or displaying a grid of devices. You are free to change things however you like! If you want to know more about how this example app is structured, please see the README file in the root of the project.

Since you do not yet have any devices, we have added a simple device creator to this application, which will walk you through setting up a device. Just click Devices on the sidenav!

Important Information

The example application makes use of a proxy API for OnePlatform, which supports standard RPC calls, as well as realtime data via WebSockets. Look at app/js/services/apiService.js for examples on how to interact with the API. There is also a Swagger spec file at https://api.exohack.io/swagger.json.

To make use of the WebSockets endpoint, you'll need to use the Phoenix Channels client contained in this project, you can see an example of it's use in app/js/directives/realtimeVoltageGraph.js, and documentation on it's API can be found here. Phoenix also supports other clients, such as iOS and Android, if that's your thing.

In order to make requests to the API, you'll need to provide the Authorization header, set with the token provided to your application in the JWTTOKEN environment variable of your container. Likewise, if you are using Phoenix, you'll need to provide {token: "$JWTTOKEN"} to the Socket constructor. This will authenticate you to the API, and automatically scope your RPC calls to the CIK for your application. You can provide another CIK in your RPC requests via the auth key of the request object, and it will take precedence, however authentication to the API will still be required via the token.
